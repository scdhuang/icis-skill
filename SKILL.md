# ICIS - 智能客户访谈系统

## 概述

ICIS 是专为销售团队设计的 AI 访谈助手，解决客户拜访中的四大痛点：
- **不知道问谁** → 自动生成访谈计划
- **不知道怎么回** → 实时话术推荐
- **不知道怎么分析** → 自动提取痛点匹配方案
- **不知道推什么** → AI 机会点推荐 + ROI 测算

## 核心能力

### 1. 访谈计划生成 (`icis plan`)
根据客户信息自动生成完整访谈计划：
- 推荐访谈对象（按优先级排序）
- 每人核心问题（3-5个）
- 预估访谈时长
- 配套材料清单

**使用方式：**
```bash
# 命令行
icis plan --company "XX公司" --industry appliances --scale "10亿/2000人"

# 或直接告诉 AI
"帮我生成 XX吸尘器有限公司 的访谈计划，家电行业，10亿规模"
```

### 2. 访谈记录分析 (`icis analyze`)
分析访谈记录，自动输出：
- 痛点提取（Top 5）
- AI 机会点匹配（Top 3）
- 业务影响量化
- Sell-in 策略生成

**使用方式：**
```bash
# 命令行
icis analyze --input interview.txt --output report.md

# 或直接粘贴记录给 AI
"帮我分析这段访谈记录：[粘贴内容]"
```

### 3. 话术查询 (`icis script`)
查询销售话术，覆盖：
- 开场白（3种角色）
- 异议处理（5种常见异议）
- Closing 话术（3种角色）
- 价值陈述（按痛点匹配）

**使用方式：**
```bash
# 命令行
icis script --type objection --context "已经有ERP了"

# 或直接问 AI
"客户说已经有 ERP 了，我怎么回应？"
```

### 4. 案例查询 (`icis case`)
查看成功案例库：
- 同行业案例
- 同痛点案例
- 完整 Sell-in 路径

**使用方式：**
```bash
# 命令行
icis case --industry appliances --pain "订单处理"

# 或直接问 AI
"有没有家电行业订单处理的案例？"
```

## 行业模板

当前支持的行业：

| 行业 | 细分 | 模板文件 |
|------|------|----------|
| 制造业 | 家电 | `templates/manufacturing/appliances.yaml` |
| 制造业 | 电子 | `templates/manufacturing/electronics.yaml` |
| 制造业 | 机械 | `templates/manufacturing/machinery.yaml` |

每个模板包含：
- 公司画像（规模、业务模式）
- 核心痛点（4-6个）
- 访谈对象（3-4人）
- AI 机会点（3-5个）
- Sell-in 策略（按角色）

## 工作流程

### 标准工作流程

```
Step 1: 输入客户信息
    ↓
Step 2: 生成访谈计划（icis plan）
    ↓
Step 3: 执行访谈（使用话术库）
    ↓
Step 4: 记录访谈内容
    ↓
Step 5: 分析访谈记录（icis analyze）
    ↓
Step 6: 输出交付物（痛点分析 + 解决方案 + Sell-in 策略）
```

### 快速工作流程

```
输入客户信息 → AI 直接输出完整方案包（5个文档）
```

## 输出文档

ICIS 生成以下交付物：

1. **访谈计划** (`{客户名}-访谈计划.md`)
   - 访谈对象安排
   - 核心问题清单
   - 时间规划

2. **痛点分析** (`{客户名}-痛点分析.md`)
   - Top 5 痛点
   - 业务影响量化
   - 优先级排序

3. **解决方案** (`{客户名}-解决方案.md`)
   - AI 机会点推荐
   - ROI 测算
   - 实施路径

4. **话术清单** (`{客户名}-话术清单.md`)
   - 开场白
   - 异议处理
   - Closing 话术

5. **行动计划** (`{客户名}-行动计划.md`)
   - 下一步安排
   - 责任人
   - 时间节点

### 示例1：生成访谈计划

```
客户：XX吸尘器有限公司
行业：制造业-家电
规模：10亿营收/2000人
业务模式：80%代工+20%自有品牌
```

**输出：**
```markdown
# XX吸尘器有限公司 - 访谈计划

## 访谈安排

### 1. IT总监（优先级1，45分钟）
**目标：** 了解系统现状、数据孤岛、AI准备度

**核心问题：**
1. 现在用哪些系统？最大的数据孤岛在哪？
2. 业务部门最抱怨系统什么？
3. 如果上AI，数据准备好了吗？

### 2. 销售总监（优先级2，45分钟）
...

### 3. 供应链总监（优先级3，45分钟）
...
```

### 示例2：分析访谈记录

**输入：**
```
李总访谈记录：
- 用SAP ERP 5年了，销售部门抱怨订单录入太麻烦
- 销售每天花2小时录入订单，经常录错，客户投诉
- 客户信息分散在Excel、微信、ERP里，查起来费劲
```

**输出：**
```markdown
# 痛点分析

## 核心痛点

### 1. 订单处理效率低 ⭐⭐⭐⭐⭐
- **客户原话：** "销售每天花2小时录入订单，经常录错"
- **业务影响：** 10个销售受影响，月损失约50单
- **匹配方案：** 智能订单处理
- **预期效果：** 2天→2小时，准确率99%+
- **ROI：** 6个月回本

## AI 机会点推荐

| 排名 | 解决方案 | 匹配度 | ROI |
|------|----------|--------|-----|
| 1 | 智能订单处理 | 95% | 850% |
| 2 | 视觉质检 | 80% | 600% |
| 3 | 智能客服 | 75% | 400% |
```

### 示例3：查询话术

**输入：**
```
客户说："我们已经有ERP了，为什么还要用飞书？"
```

**输出：**
```
回应话术：
"ERP解决的是流程问题，飞书+AI解决的是效率问题。

```

## 扩展开发

### 添加新行业模板

1. 复制现有模板：
   ```bash
   cp templates/manufacturing/appliances.yaml templates/manufacturing/retail.yaml
   ```

2. 修改内容：
   - 行业名称
   - 细分行业
   - 公司画像
   - 核心痛点
   - 访谈对象
   - AI 机会点
   - Sell-in 策略

3. 验证模板：
   ```bash
   icis validate --template templates/manufacturing/retail.yaml
   ```

### 添加新案例

编辑 `knowledge_base/cases.json`，添加案例：
```json
{
  "id": "CASE_002",
  "company": "YY电子科技有限公司",
  "industry": "制造业-电子",
  "scale": "5亿/800人",
  "pain": "质量追溯困难",
  "solution": "视觉AI质检+数据溯源",
  "result": "追溯时间从4小时降到5分钟",
  "contact": "质量经理王经理",
  "sell_in_path": "产线试点→效果对比→全面推广",
  "tags": ["制造业", "电子", "质检", "质量经理决策"]
}
```

## 注意事项

1. **模板匹配**：如果客户行业不在模板中，选择最接近的行业
2. **话术调整**：话术库提供标准版本，根据实际情况调整
3. **ROI 测算**：默认使用行业平均值，根据客户实际情况调整
4. **案例引用**：引用案例时脱敏处理，避免泄露客户信息
